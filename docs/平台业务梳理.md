# 天府未来教育中心 · 业务梳理

> 课后延时服务平台。天府通提供**流量入口和支付通道**，平台自建系统，负责学校、场地、机构、老师、课程、班级、报名、销课、售后和结算的全链路管理。

---

## 一、平台定位（一句话）

**把审核过的校外教培机构课程，精准引入校内课后延时时段（17:00–18:30），家长一次付费、资金平台托管、机构按课时挣钱、平台按月分账结算。**

---

## ⚑ 0707 会议首期决议（与下文有出入时以此为准）

- **付费模式**：首期只做「**先付后学**」（家长报名即付全款进监管账户），暂不做「先学后付」。机制不变：资金托管 + 按课时销课释放。
- **销课/确认**：**按课时逐节确认**——家长确认一节即消课一节、计入当月结算；有异议单节暂停结算；3 天不确认自动确认。已去掉整体结课评语，仅保留逐节确认 + 老师标注考勤。
- **考勤**：精简为 **已到 / 缺勤** 两项（老师标注，同步家长端）；考勤与课堂记录为**非强制辅助记录**，与销课无强制先后。
- **学校端**：**首期不单独上线**，场地维护 / 排课确认由平台运营在「平台管理后台」代做，学校侧线下配合（`/school/` 为远期规划演示）。
- **结算**：按月，平台生成**对账单**经机构确认后提交天府通分账；向天府通下发「**订单详情 + 对账单**」两份独立数据（不合并）。
- **分账**：**四方**（机构 / 平台 / 地方国企 / 天府通通道），**不同课程比例不同，在课程审核时单独配置**；「学校分成」措辞改为「**学校服务费**」。
- **课程配置**：增加「报名开始 / 报名截止 / 开课时间」；场地区分「校内 / 校外」。
- **退费**：首期**不做自动化退费**，仅记录，规则由运营与学校线下协商。
- **称谓**：后台侧「老师」统一为「**教师**」（人名如“王老师”保留）。

> 待确认（D 组开放点）：① 老师标注「缺勤」的课时是否仍需家长确认/是否扣费；② 请假退费具体规则；③ 与天府通的数据回传方式与分账比例。

## 二、五个端的定位与职责

| 端 | 使用者 | 一句话定位 | Demo 地址 |
|---|---|---|---|
| 家长端（H5） | 学生家长 | 选课报名、缴费托管、看结果、确认结课 | `/` |
| 平台管理后台 | 平台运营/审核/财务 | 全链路总控：审核、分发、监管、结算 | `/admin/` |
| 机构端 | 机构管理员 | 管老师、发课程、盯销课、看结算 | `/org/` |
| 老师端 | 机构老师（子账号） | 到校签到、提交课堂记录、写结课评价 | `/org/`（切角色） |
| 学校端 | 学校管理员 | 管本校场地、确认排课、确认上课事实 | `/school/` |

### 1. 家长端（天府通入口 / H5）
**做什么：**
- 自动识别孩子所在学校，只看到**本校被分发的课程**（多孩子可切换）
- 选课程 → 选**具体班级**（时段 SKU，满员不可选；时间冲突提醒，家长确认后可继续）→ 一次性缴费（微信/天府通支付，资金进**平台监管账户**）
- 查看报名状态（待成班/上课中等）；上课后在学习成果页**逐节确认课时**（考勤为老师标注：已到/缺勤），确认即消课；对某节有异议可标注（暂停该节结算）；3 天不确认自动确认
- 有问题发起售后

**不做什么：** 不选老师（老师是课程附属）、不约课（固定班次模式）

### 2. 平台管理后台（总控）
**做什么：**
- **建底盘**：创建学校档案、维护学校场地
- **守门**：机构入驻审核、老师资质审核、课程内容审核（机构和课程**分开审**，重点防内容风险）
- **核心动作——课程分发**：从课程库选课 → 配置到指定学校（班级/时间/场地/价格/名额可按校不同）→ 只有被分发学校的家长可见
- **监管**：订单与退款、销课确认、售后仲裁、操作日志
- **管钱**：按月生成机构结算单（销课累计 − 平台服务费 − 学校分成 − 退款扣减）→ 财务审核 → 打款

**关键权力：** 课程库里的课不直接面向家长，**分发权在平台**；钱在平台监管账户，**结算权在平台**

### 3. 机构端
**做什么：**
- 维护机构资料与结算账户（入驻需平台审核通过）
- **创建老师**（资质提交平台审核，通过才能带班）
- **发布课程**（提交平台审核，通过进课程库；机构不能直接上架给家长）
- 查看被分发形成的班级、学生名单、上课/销课情况
- **看钱**：月度结算单、销课明细、机构实收

**不做什么：** 不直接触达家长销售、不碰托管资金

### 4. 老师端（机构子账号，权限隔离）
**做什么：**
- 只看**自己负责的班级**和学生名单
- 到校**上课签到**（已到/迟到/请假/缺勤）→ 提交**课堂记录**（内容/课堂情况/随堂评语）→ 触发销课流程
- 班级结课后提交**结课评价**（家长端学习成果页展示的那份）

**看不到：** 机构结算、订单金额、任何财务信息

### 5. 学校端
**做什么：**
- 维护**本校场地**（类型/容量/可用时段，容量直接限制班级名额）
- **确认本校课程开放**（平台分发后学校点头，家长端才可见）
- 查看本校报名名单（手机号脱敏）
- 班级成班后**确认排课**（教室/时间，有冲突可调整或退回平台）
- 查看老师签到与上课记录，**确认上课事实 / 标记异常**（销课确认的一环）
- 协同售后（场地调整、时间冲突、转班可行性等本校事项）

**不做什么：** 不审核机构和课程资质（平台审）、**不见任何结算金额**（只确认"课上没上"，不管"钱怎么分"）

## 三、整体业务流程（四个阶段）

### 阶段一：搭底盘（平台主导）
```
平台创建学校档案 + 学校维护场地
→ 机构申请入驻 → 平台审核（营业执照/法人/协议）
→ 机构创建老师 → 平台审核资质
→ 机构发布课程 → 平台审核内容 → 进入课程库
```

### 阶段二：分发与招生
```
平台把课程库课程分发到指定学校（一课可多校，各校配置可不同）
→ 学校确认开放 → 该校家长端可见
→ 家长选班级报名 → 一次性全额缴费 → 资金进平台监管账户（托管）
→ 达到最低成班人数 → 成班 → 学校确认教室与排课
→ 未成班：自动全额退款并通知家长
```

### 阶段三：上课与销课（核心循环，每节课一遍）
```
老师到校签到 → 上课 → 提交课堂记录（出勤/内容/评语）
→ 生成一条销课记录（已上课待确认）
→ 学校/平台确认上课事实 → 已确认销课（本节费用进入可结算范围）
→ 有问题 → 标记异常 → 暂停计费 → 机构 2 个工作日内说明 → 平台复核
```

### 阶段四：结课与结算（按月）
```
班级结课 → 老师提交结课评价 → 家长端查看学习成果
→ 家长确认（3 天不确认自动确认）
→ 月底平台生成机构结算单：
   应结算（当月已确认销课合计）
   − 平台服务费（约 10%）
   − 学校分成（规则预留，约 5%）
   − 退款扣减
   = 机构实收
→ 财务审核 → 确认打款 → 已结算
```

## 四、资金流转（汇报重点）

```
家长一次付 800 元（10 节课）
        ↓
   平台监管账户（托管，机构一分钱拿不到）
        ↓  每上完一节、确认一节
   机构累计 1 节可结算（80 元/节 × 实到规则）
        ↓  月底
   结算单：销课累计 − 服务费 − 学校分成 − 退款 = 机构实收
        ↓
   打款到机构对公账户
```

**这套设计解决三个信任问题：**
1. 家长怕机构跑路 → 钱在平台，上一节结一节
2. 机构怕家长赖账 → 先收款再上课，钱是确定的
3. 平台/学校要监管 → 销课确认权在平台和学校手里，虚假上课结不了账

## 五、关键规则速查

| 规则 | 内容 |
|---|---|
| 成班 | 满最低人数开班；满员停止报名；未成班自动退款 |
| 班次 | 固定班固定时段固定老师（不做约课/走班），一门课可多班次，家长报的是班级 |
| 时间冲突 | 家长已报课程时段冲突的班次禁止再报 |
| 结课确认 | 家长 3 天未确认 → 系统自动确认（防流程卡死） |
| 销课异常 | 暂停计费，机构 2 个工作日内说明，平台复核后计费或作废 |
| 售后时限 | 转机构处理限时反馈，超时平台介入；退款从机构结算扣减 |
| 老师管理 | 机构管人、平台审资质；前台重课程轻个人，固定老师带完整期 |

## 六、Demo 演示动线建议（给领导演示）

1. **平台后台**：看板讲业务闭环流程条 → 审核一家机构 → 审核课程入库 → **分发向导**（一门课勾选多所学校）
2. **学校端**：确认课程开放 → 确认排课 → 演示"学校不见钱"
3. **家长端**（手机比例）：选班（看满员/冲突禁选）→ 微信支付托管 → 学习成果与确认
4. **机构端**：切老师角色演示权限隔离（无金额列）→ 签到 → 课堂记录 → 切回管理员看销课与**结算单明细**
5. 收尾回到平台后台**结算管理**：审核 → 确认打款，闭环完成
